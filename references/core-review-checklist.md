# 通用评审清单

## 1. 架构与 SOLID

- SRP：类或方法不要混入无关职责
- OCP：新增变体时，不应必须高风险修改大量既有分支逻辑
- LSP / ISP / DIP：关注继承滥用、接口过宽、业务逻辑直接绑定具体 IO / 网络 / 存储实现
- 关注 controller 直接处理持久化规则、绕过 service、业务逻辑泄漏到 common/util 层

## 2. 安全与可靠性

- 鉴权/鉴权缺口、权限校验缺失、租户/归属校验缺失
- SQL 注入或不安全的动态 SQL 拼接
- 密钥泄漏、敏感日志、token 暴露
- 外部调用缺少超时、兜底或异常处理
- 并发竞争、check-then-act、lost update、事务边界缺失
- DB / cache / message / 外部副作用写入不一致

## 3. 代码质量

- 吞异常，或无正当理由使用 `catch (Exception)`
- 缺少 null、空集合、数值边界处理
- 返回结构不一致
- 请求边界缺少必要校验
- 魔法值、魔法字符串、重复逻辑

## 4. 性能与数据访问

- N+1 查询
- 可批量却逐条查的查询
- 高风险全表扫描或缺少查询边界
- 大对象、大结果日志
- 不必要的重复远程调用
- 缓存用法破坏正确性，或缺少失效策略思考

## 5. 可删除代码

- 不可达代码
- 已废弃分支逻辑
- 重构后的重复路径
- 过时兼容代码

删除建议要区分：

- 可以立刻删
- 需要迁移或验证后再删
